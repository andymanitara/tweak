Class: tweak.EventSystem

Defined in: app\lib\events.coffee

Overview

The framework is driven by a powerful and simple event system.

The event system simply stores the callbacks in an object tree. The tree allows quick traversal of events, during interaction with the event API

# (Boolean) on(context, name, callback, maxCalls = null)

Add an event listener into the event system. The event system will store the object based on its event name

Examples:

Sample Event (JS) 

tweak.Events.on(this, "sample/event", function(){
  alert("Sample event triggered.")
})

Sample Event (CoffeeScript) 

tweak.Events.on @, "sample/event", ->
  alert "Sample event triggered."

Sample Event with Max Calls (JS) 

// This example will allow the event to be called twice
// The event is automatically removed from the event object on max calls
tweak.Events.on(this, "sample/event", function(){
  alert("Sample event triggered.")
}, 2)

Sample Event (CoffeeScript) 

// This example will allow the event to be called twice
// The event is automatically removed from the event object on max calls
tweak.Events.on(@, "sample/event", ->
  alert "Sample event triggered."
,
2)

Parameters:

  • context ( Object ) — The contextual object of which the event to be binded to
  • name ( String ) — The event name; split on the / and : characters
  • callback ( Function ) — The event callback
  • maxCalls ( Number ) — Default=null. The maximum calls allowed on the event listener

Returns:

  • ( Boolean ) — Returns true if added, or false if not added

# (Boolean) off(context, name, callback)

Remove from events by given name

Parameters:

  • context ( Object ) — The contextual object of which the event is binded to
  • name ( String ) — The event name; split on the / and : characters
  • callback ( Function ) — (optional) The callback function of the event. If no specific callback is given then all the events under event name are removed

Returns:

  • ( Boolean ) — Returns true if event(s) are removed, or false if no event are removed

~ (Boolean) trigger(obj, params)

Trigger events by name and context

Parameters:

  • obj ( Object ) — {name:String (name of the event), context:Object (context of the event)}
  • params ( ... ) — Params to pass into the callback function

Returns:

  • ( Boolean ) — Returns true if event is triggered and false if nothing is triggered

# (Event, Null) find(name, build = false)

Iterate through the events to find given event

Parameters:

  • name ( String ) — The event name; split on the / and : characters
  • build ( Boolean ) — Default = true. If set to true the event object will be built by the name if can be found.

Returns:

  • ( Event, Null ) — If event object is found/created then it is returned.

# (void) reset()

Resets the events back to empty.
