Options Hash: (options):

  • speed ( Number ) — The amount of time per check in ms
  • quiet ( Boolean ) — If true then it wont trigger events

# (void) stop()

Stop watching the router for changes

# (void) check(quiet = false)

Note: event name is made up from the data in the url. The router data is split up by / \ per data set, and by : = between the key and data.

Check the window location, if there is an update from previous url then trigger an event

Examples:

hash url examples 

tweakjs.com/#search/safe/version=2
triggers:
  #{@name}:router:data:search
  #{@name}:router:data:safe
  #{@name}:router:data:version (passes in 2)
  #{@name}:router:changed (passes in {search:true, safe:true, version:2})

tweakjs.com/#version:1/search/safe/version=2
triggers:
  #{@name}:router:data:version (passes in 1)
  #{@name}:router:data:search
  #{@name}:router:data:safe
  #{@name}:router:data:version (passes in 2)
  #{@name}:router:changed (passes in {search:true, safe:true, version:2})

Parameters:

  • quiet ( Boolean ) — If true then it wont trigger events

Events:

#{@name}:router:data:#{data

key} Triggers an event based on the key value and if there is any data attached to the router key then that data is passed through aswell. Triggered for each data set

#{@name}:router:changed

Triggers an event and passes the data of the url back

# (void) set(obj, quiet = false)

Set the url hash with certain data; triggering router based events

Parameters:

  • obj ( Object ) — Simple object to pass into url. Can't be more than one level deep.
  • quiet ( Boolean ) — If true router events will not be triggered on change
