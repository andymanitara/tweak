Class: tweak.Model

Defined in: app\lib\model.coffee

Overview

The model is simply a way of storing some data, with event triggering on changes to the model In common MVC concept the Model is not always a database. So the controller should be used to get data from a database. The controller is normally the interface between the view and the models data. When the model updates it will fire of events to Event system; allowing you to listen to what has been changed. The controller can then detirmine what to do when it gets updated. You can update the model quietly aswell.
